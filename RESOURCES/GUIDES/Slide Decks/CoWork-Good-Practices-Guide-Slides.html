Working smarter

When to Start Fresh

CoWork reads your About Me files at session start, giving you a clean briefing. Longer sessions spread attention across more context — which can be useful, but it also means Claude's focus gets divided.

New session when:

  • Switching to a completely different task
  • Current session feels sluggish or unfocused
  • You've updated About Me files or global instructions
  • Finished a major piece of work

Continue when:

  • Iterating on something Claude just produced
  • You need earlier context from the conversation
  • Mid-way through a multi-step task

Good rule of thumb: new task, new session. Memory means you never lose context between sessions.

Better prompts

Give Claude the Full Picture

What you want

The actual deliverable. "Draft three email subject lines for the launch" beats "help me with an email."

Why it matters

Context that shapes the approach. "This client is price-sensitive" changes what Claude produces.

What good looks like

Your standards. "Keep it under 200 words" or "match the tone of the last newsletter."

For anything big, ask Claude to make a plan first: "Before you start, outline the steps and check if you need anything from me."

The secret weapon

Memory Makes It a Living System

Good entries

Decisions, discovered preferences, system changes, important context. Specific enough that a future session can act on them.

Skip these

Routine completions, minor tweaks, anything that won't matter next week.

When it gets stale

System Review catches this. If you spot something obviously wrong, just tell Claude to update it.

The feedback loop

One correction + one memory entry = permanent improvement. Every specific fix you give Claude gets logged and applied automatically.

After big sessions, ask Claude: "Have you updated the memory files?" A quick debrief before closing ensures nothing important slips through.

The fastest path

The Art of Feedback

Vague feedback helps a little. Specific feedback helps a lot. One well-phrased correction can permanently change how Claude writes for you.

Vague

"This doesn't sound right" — Claude will try again but might miss the point.

Specific

"Too formal — I'd never say 'I trust this finds you well.' Use 'Hey' and jump straight to the point." — Claude logs this and never makes the same mistake.

If you make the same correction more than twice, it belongs in your writing rules or voice profile, not just memory.

Keeping it clean

File Hygiene

About Me files

Update when something meaningful changes. Don't update for every small thing — that's what memory is for.

CLAUDE.md

Keep it lean. Every instruction competes for attention. Structural rules, not detailed content.

Projects

Leave finished folders in place — the memory log is valuable for future similar work.

Templates

Spot a repeating pattern? Make a template. Even a rough structure beats starting from scratch.

Troubleshooting

When Claude Gets Stuck

  • Rephrase the request. Saying the same thing louder won't help — come at it from a different angle.
  • Break it down. "Write my entire website copy" will struggle. "Write the hero section headline" will work.
  • Start a fresh session. Memory means you won't lose context. A clean reload fixes most sluggishness.
  • Give more context, not less. Generic output usually means not enough information to be specific.
  • Check the basics. Ask Claude: "Do a system health check" — it reads all your files and reports what it sees.

Good to know

Capabilities and Limits

Drafts, not sends

Emails, Slack messages, calendar events — Claude creates drafts. You always review before anything leaves your account.

App needs to be running

Scheduled tasks run through the desktop app. Toggle "keep computer awake" in settings. CoWork OS checks at session start if anything's overdue.

Folder scope

Claude sees files inside your workspace folder. Nothing else on your computer unless you give access.

Dedicated desk setup

For power users: a computer that stays on at home. Scheduled tasks run reliably, and Dispatch lets you control CoWork from your phone wherever you are.

An important choice

Why We Use One Folder

CoWork has a native Projects feature in the sidebar — separate workspaces with their own instructions and memory. You might wonder whether to use those instead of this folder structure.

The problem with splitting

When you link a CoWork Project to a specific folder, Claude can only see files inside it. Your global instructions still apply — but if they tell Claude to read your About Me files and those files are outside the project folder, Claude can't reach them. The instruction travels everywhere. The file access doesn't.

What one folder gives you

CoWork OS keeps everything under one roof. Your voice, your writing rules, your project briefs, your memory — all accessible from any conversation. The WORK AREAS/ structure gives you per-project organisation without losing the shared foundation.

Select your entire CoWork OS folder as your workspace. Use the WORK AREAS/ structure for organising individual pieces of work. You get project separation and shared context at the same time.
